Introduction to HTML

October 2020

HTML stands for Hypertext Markup Language. Initially launched in 1993 and currently on v5, it is the main markup language used for webpage building. It is developed and maintained by the World Wide Web Consortium (W3C) and the Web Hypertext Application Technology Working Group (WHATWG). There are two standards (from the two different groups) but both are quite similar.

In order to start creating with HTML, you need the following:

  • Computer (you probably have one if you're reading this)
  • Text editor to write the code (I use Visual Studio Code for my coding projects but there are many good ones out there)
  • Web browser (to visualize your progress)

Create a Webpage and View in a Browser

Creating a webpage is very easy (you can create one in less than two minutes!). Follow these steps:

  1. Open your text editor of choice and create a new file called index.html. Be sure the file extension is .html because that's how the browser identifies the file as an html.
  2. Copy the following code into the file:
    3. <!DOCTYPE html>
    <html>
    <head>
    <title>My First HTML Page</title>
    </head>
    <body>
    <h1>My First Page</h1>
    <p>My first page in less than 2 minutes!</p>
    </body>
    </html>
  3. Launch your browser and open up the file to see what you've created!

Elements

HTML documents are composed of various elements that make up the page, and tags are used to define each of those elements. We can visit our previous code snippet and break it apart:

  • !DOCTYPE html: a declaration that tells the browser which version of HTML you're using
  • html: this is considered the root elements because all other elements are contained within it
  • head: this element contains information about the webpage that the user typically does not see (such as meta tags, CSS file location, etc.). One notable exception is the Title attribute, which is displayed as the tab title at the top of a browser
  • body: this contains the bulk of viewable material where you'll produce all your fantastic creations!

You may have noticed the p and h1 tags as well. We'll take a look at those later. You may have also noticed that each tag has a subbsequent closing tag with a /. This is very important as it tells the browser the content of that element is over.


Attributes

Attributes provide more information about the aforementioned elements. They are extremely important because attributes can define how the tag should behave, how it should be referenced in CSS stylesheets, a hyperlink that the text should direct to, etc. Attributes are always in the start tag of an element and follow the format name = "value". You can also stagger multiple attributes into the same element. A list of all global attributes can be found here. Below is a list of commonly used attributes to start off:

href

A hyperlink can be inserted via the <a> tag, but the actual link lives in the href attribute.

<a href="www.google.com">Google</a>

src

The <img> element is used to embed an image into a page. The path of the image is specified via the src attribute.

<img src="img_dog.png">

Notice that there is no end tag above; image elements only have a beginning tag.

style

The style attribute is what's used to give some flair to the text and add some styling (discussed more below). It can accommodate all the style guides used in CSS and applies only to the element that it's applied on.

<p style="color: blue;">A blue paragraph</p>

Semantics

You may have heard of CSS, which is used to give your webpages personality (and which I'll hopefully cover in another article). HTML however offers some pre-configured ways to format content without the need for CSS.

Headings

There are six heading levels built-in with HTML: h1 to h6, in decreasing priority. Headings provide an easy way to create visual hierarchy to direct the reader's attention to the most important elements first. You've already used one, but here they are in practice:

Heading 1

Heading 2

Heading 3

Heading 4

Heading 5
Heading 6

Headings can be used through tags, such as below.

<h1>Hello there</h1>

Formatting

There are also specific ways to format your content to call out attention to any text:

<i> - italics text

<em> - emphasized text

<b> - bold text

<strong> - strong/important text

<sub> - subscript text

<sup> - superscript text

<del> - deleted text

<mark> - marked text

<u> - underlined text

Any of the above modifications can be used through elements:

<p>Peaches are <i>delicious</i>.</p>

An interesting thing to note here is the seemingly bizarre distinction between i and em, and b and strong. There are coding standards that HTML uses that may not necessarily create visible differences to the user, but instead serve to distinguish text that is truly important (strong/em) versus text that simply needs to be called out (b/i). HTML standards can be found at the WHATWG page here.

Lists

There are two main types of lists that HTML can accommodate for: ordered (numbered) and unordered (bulleted) lists; however, the syntax is quite similar. Ordered lists can be invoked through the <ol> tag; unordered lists can be triggered through the <ul> tag. Once initiated however, both types of lists use the <li> element to instantiate each list item.

<ol>
<li>Ordered</li>
<li>List</li>
</ol>

<ul>
<li>Unordered</li>
<li>List</li>
</ul>

Styling

HTML provides content, but it doesn't change the look and feel of the site; that's where CSS comes into play. CSS stands for Cascading Style Sheets and it's a way to apply style conventions on the webpage. It's extremely intuitive. For example, to set the color and size of some text, you can simply do the following:

color = green;
font_size = 15px;

A full list of all styling variables can be found here (there's a lot!).

There are three mainstream ways to apply styles: inline, embedded, and external.

Inline

Inline refers to the style guide being applied directly on the element in question, hence it's "inline" with the element. This is a fast and convenient way to apply adhoc styling guides, and I mainly find myself using them when I want to apply a bespoke style guide to an element (such as with the heading examples above - my headings have default color black, but I wanted to make those grey). However, inline styles should be used sparingly, as they can spiral into an extremely disorganized and difficult-to-change system.

<p style="color=green; font-size=15px;"> Inline styling </p>

Notice the use of semicolons and quotations to contain the style variables.

Embedded

Embedded style sheets are a level up when it comes to maintainability. Although the styling nouns stay the same, all of them are assembled in the head element of the HTML page. This allows you to quickly and exhaustively alter the entire page's style without much effort.

<!DOCTYPE html>
<html>
<head>
<title>My First HTML Page</title>
<style>
body {
color = green;
font-size = 15px;
}
</style>
</head>
<body>
...

There are a couple things to note here:

  • The body tag in the head refers to the body element below, meaning the styling that's created for body in the head will apply to the entire body element and everything that's in it. But what if we want more specific styling for different elements? That's ok! Styles are automatically overriden by more granular styling details.
  • The body element is specified. CSS is extremely flexible and you can reference any element by their "name". For example if I create an element with a class attribute of ananyas-html-course then I can use that in the styling guide to style all the elements within ananyas-html-course.

External

External styling docs are very similar to embedded, except that they live in a .css file of their own that you can directly call in the head. More info in the CSS introductory post that I'll hopefully get to soon.
